LAMMPS submission package - file guide

01_input_files
- solvent_mix_1.data: original converted LAMMPS data file from GROMACS/Packmol model.
- solvent_mix_1.minimized.data: minimized structure from the first minimization.
- solvent_mix_1_26ps.data: stable 26 ps structure used as restart point after direct NPT was too aggressive.
- pair_coeffs.in: Lennard-Jones pair coefficients converted from GROMOS/ATB C6/C12 parameters.

02_run_scripts
- in.minimize: LAMMPS minimization script.
- in.equil_npt: original direct NPT script; kept for record, but it was too aggressive for the low-density starting box.
- in.safe_nvt_26ps: safer NVT stabilization from the 26 ps stable frame.
- in.safe_npt_26ps: safer slow NPT equilibration script.
- convert_gromacs_to_lammps.py: Python conversion script.
- run_lammps_from_python.py: Python subprocess workflow for launching LAMMPS.
- run_lammps_api.py: Python API workflow using `from lammps import lammps`.
- run_safe_pipeline.ps1: PowerShell pipeline used to run NVT then NPT.

03_output_files
- current_in_progress_180k: current output snapshot copied while NPT was still running.
- When the NPT run reaches 200000 steps, final files should be copied here:
  * safe_npt_26ps.log
  * safe_npt_26ps.lammpstrj
  * solvent_mix_1_26ps_npt_100ps.data
  * final thermo CSV and summary.

04_visualization
- VMD2-compatible XYZ and Tcl files.
- Open vmd_safe_npt_current.xyz or source view_safe_npt_current.tcl in VMD2.

05_report_assets
- CSV/PNG/summary files for report writing.
- These are current snapshots and can be regenerated after the final 200000-step run completes.

06_notes
- Notes and status files.

Simulation method summary for report:
LAMMPS was used for molecular dynamics simulations. Python scripts were used for format conversion, LAMMPS API/subprocess control, output archiving, and post-processing. VMD2 was used for trajectory visualization.
